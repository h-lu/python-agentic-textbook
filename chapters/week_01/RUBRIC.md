# Week 01 评分标准

## 评分维度

### 1. 功能完成度（40%）

| 等级 | 分值 | 标准 |
|------|------|------|
| 优秀 | 36-40 | 基础功能完整，进阶功能完成 2 项以上，代码运行流畅无 bug |
| 良好 | 28-35 | 基础功能完整，进阶功能完成 1 项，有小瑕疵但不影响使用 |
| 合格 | 20-27 | 基础功能基本完整，能运行但输出格式有瑕疵 |
| 不合格 | 0-19 | 基础功能缺失或无法运行 |

**检查点：**
- [ ] 能使用 `input()` 正确获取用户输入
- [ ] 能使用 `print()` 输出名片
- [ ] 使用了 f-string 进行格式化
- [ ] 名片格式整齐，有边框或分隔线
- [ ] （进阶）多语言切换功能正常
- [ ] （进阶）输入验证逻辑正确

---

### 2. 代码质量（30%）

| 等级 | 分值 | 标准 |
|------|------|------|
| 优秀 | 27-30 | 变量命名清晰规范，代码结构清晰，有适当的空行分隔逻辑块 |
| 良好 | 21-26 | 变量命名基本规范，代码结构合理，可读性较好 |
| 合格 | 15-20 | 变量命名能看懂，代码能运行但结构较乱 |
| 不合格 | 0-14 | 变量命名混乱（如单字母变量），代码难以阅读 |

**检查点：**
- [ ] 变量名有意义（如 `name`、`age`，而不是 `a`、`b`）
- [ ] 变量名使用小写字母和下划线（snake_case）
- [ ] 代码缩进正确（4 个空格）
- [ ] 逻辑块之间有适当的空行
- [ ] 没有未使用的变量

**常见扣分项：**
- 使用拼音缩写（如 `xm`、`nl`）作为变量名（-3 分）
- 变量名大小写混用（如 `Name`、`userName`）（-2 分）
- 一行代码过长（超过 80 字符）（-1 分）

---

### 3. 文档与注释（15%）

| 等级 | 分值 | 标准 |
|------|------|------|
| 优秀 | 13-15 | 文件头有注释说明，关键步骤有注释，注释简洁有效 |
| 良好 | 10-12 | 有文件头注释，主要逻辑有注释 |
| 合格 | 7-9 | 有少量注释，能说明代码用途 |
| 不合格 | 0-6 | 完全没有注释，或注释全是废话 |

**检查点：**
- [ ] 文件头部有注释说明程序功能和作者
- [ ] 关键逻辑有简洁的注释
- [ ] 没有显而易见的废话注释（如 `# 打印输出` 紧跟 `print()`）

**标准文件头示例：**
```python
# business_card.py
# 个人名片生成器
# 功能：获取用户输入并生成格式化的个人名片
# 作者：你的名字
# 日期：2026-02-08
```

---

### 4. 测试通过（15%）

| 等级 | 分值 | 标准 |
|------|------|------|
| 优秀 | 13-15 | 所有 pytest 测试通过，包括基础测试和边界测试 |
| 良好 | 10-12 | 主要测试通过，个别边界情况未处理 |
| 合格 | 7-9 | 基础功能测试通过，进阶功能测试未通过 |
| 不合格 | 0-6 | 基础测试未通过 |

**测试项目：**

1. **基础功能测试**
   - 程序能正常运行不报错
   - 能正确处理标准输入
   - 输出包含所有必需字段

2. **格式测试**
   - 使用了 f-string（检查代码中是否有 `f"` 或 `f'`）
   - 输出有边框或分隔线

3. **代码规范测试**
   - 没有使用单字母变量（`a`, `b`, `i` 等作为非循环变量）
   - 没有变量名大小写错误（`Name` vs `name`）

## 评分示例

### 示例 A：优秀作业（95 分）

```python
# business_card.py
# 个人名片生成器
# 功能：交互式生成美观的个人名片
# 作者：李小明

print("=== 名片生成器 ===\n")

# 获取用户输入
name = input("请输入姓名：")
age = input("请输入年龄：")
job = input("请输入职业：")

# 验证年龄输入
while not age.isdigit():
    print("年龄必须是数字！")
    age = input("请输入年龄：")

# 格式化输出
print(f"\n╔{'═' * 30}╗")
print(f"║{'个人名片':^30}║")
print(f"╠{'═' * 30}╣")
print(f"║  姓名：{name:<20}║")
print(f"║  年龄：{age + ' 岁':<20}║")
print(f"║  职业：{job:<20}║")
print(f"╚{'═' * 30}╝")
```

**评语：** 基础功能完整，进阶功能（输入验证、ASCII 边框）实现良好，变量命名规范，代码结构清晰。

### 示例 B：合格作业（75 分）

```python
# business_card.py

name = input("请输入姓名：")
age = input("请输入年龄：")
job = input("请输入职业：")

print("==============================")
print("        个人名片")
print("==============================")
print("姓名：" + name)
print("年龄：" + age + " 岁")
print("职业：" + job)
print("==============================")
```

**评语：** 功能完整，代码能正确运行。但有以下改进空间：建议使用 f-string 替代字符串拼接（更简洁易读）；缺少文件头注释；没有进阶功能；变量命名尚可但可以更具描述性；代码结构可以增加空行分隔。总体来说是合格的入门级作业。

### 示例 C：不合格作业（50 分）

```python
business_card.py

a = input()
b = input()
c = input()

print(a)
print(b)
print(c)
```

**评语：** 变量命名完全无意义（-15 分），没有提示信息用户不知道输入什么（-10 分），输出完全没有格式（-10 分），缺少注释（-5 分），没有使用 f-string（-5 分），没有边框（-5 分）。
