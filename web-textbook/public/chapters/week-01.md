## 1. 让 Python 说 "Hello"

程序是什么？

说白了，程序就是一系列指令的集合。你写下一行命令，计算机执行它，然后执行下一行。就像给一个非常听话但有点死脑筋的助手写操作手册——它不会猜你想做什么，只会严格执行你写的每一个字。

Python 里最简单的指令，是让计算机说点什么。打开你的编辑器，新建一个文件，输入这一行：

```python
print("Hello, World!")
```

保存为 `hello.py`，然后在终端里运行：

```bash
python3 hello.py
```

你会看到屏幕上跳出：

```
Hello, World!
```

这就是你的第一行 Python 代码。全世界学编程的人几乎都是从这句话开始的，它像是一种仪式——标志着从今天开始，你可以让计算机按照你的指令做事了。

`print()` 是一个**函数**（function），你可以把它理解成"让 Python 做某事的命令"。括号里是你想让 Python 显示的内容。注意那些引号——它们告诉 Python：这是一段文字，不是指令。

`print()` 其实比你想象的更灵活。它可以一次输出多个东西，自动用空格隔开：

```python
print("姓名：", "张三")
print("年龄：", 25)
```

输出：

```
姓名： 张三
年龄： 25
```

每一行 `print()` 默认会换行。如果你不想换行，可以用 `end` 参数：

```python
print("加载中", end="")
print("...", end="")
print("完成！")
```

输出：

```
加载中...完成！
```

好了，现在我们来做一个稍微实用点的东西。假设你要去参加一个技术交流会，需要一张简单的个人名片。先用最朴素的方式——直接输出：

```python
print("========== 个人名片 ==========")
print("姓名：李小明")
print("职业：Python 学习者")
print("邮箱：xiaoming@example.com")
print("==============================")
```

运行一下。嗯，像那么回事了。虽然信息是写死的，但至少格式整齐。

这就是我们这周的起点：一张硬编码的静态名片。接下来，我们要让它变得越来越智能。

## 2. 让程序记住东西

上一节的名片有个问题：如果你想帮朋友也打印一张名片，就得改代码里的每一个信息。张三改成李四，程序员改成设计师……改来改去很麻烦。

如果程序能"记住"这些信息，然后反复使用，该多好？

这就是**变量**（variable）的作用。你可以把变量想象成贴标签的盒子——盒子里装着数据，标签上写着名字。以后你想用那个数据，直接喊标签的名字就行。

```python
name = "李小明"
job = "Python 学习者"
email = "xiaoming@example.com"

print("========== 个人名片 ==========")
print("姓名：", name)
print("职业：", job)
print("邮箱：", email)
print("==============================")
```

看到那个等号了吗？在数学里 `=` 表示"等于"，但在 Python 里它表示"把右边的值存到左边的名字里"。这个动作叫**赋值**（assignment）。

第一次见确实有点别扭，但写几次就习惯了。

现在如果你想帮朋友打印名片，只需要改前三行，后面的输出代码完全不用动。这就是变量的威力——把"会变的东西"和"不变的逻辑"分开。

小北照着教材敲完这段代码，自信满满地按下回车——然后 Python 用红色大字教育了他：

```
NameError: name 'Name' is not defined
```

"等等，我明明定义了 `name` 啊？"小北盯着屏幕，反复检查了三遍。

你发现问题了吗？他写的是：

```python
name = "李小明"
print("姓名：", Name)   # 注意这里的大写 N
```

`name` 和 `Name` 在 Python 看来是两个完全不同的名字——就像"张伟"和"张威"在户口本上是两个人。Python 不会猜你"大概是想说那个 name 吧"，它会直接告诉你"我不认识 Name"。

所以养成一个习惯：**变量名全部用小写字母**，如果需要分隔单词就用下划线（比如 `user_name`）。这样永远不会在大小写上翻车。

变量名是你给数据取的标签，可以叫 `name`，可以叫 `xingming`，甚至可以叫 `abc123`——只要不用数字开头、不包含空格和特殊符号就行。但好的变量名应该让人一眼看懂里面存的是什么。

```python
# 好的变量名
user_name = "李小明"
user_age = 25

# 糟糕的变量名（虽然能运行，但看不懂）
x = "李小明"
y = 25
```

现在你的名片已经"可配置"了。但还有一个问题：每次换人都要改代码，能不能让程序运行时直接问用户？

## 3. 跟用户聊起来

变量让名片可配置，但改代码还是太麻烦。能不能运行时问用户？

当然可以。Python 有一个内置函数叫 `input()`，它会暂停程序，等用户输入点什么，然后按回车：

```python
name = input("请输入你的姓名：")
print("你好，", name, "！")
```

运行这段代码，你会看到程序停在那里，光标闪烁。输入"李小明"，按回车，程序就会跟你打招呼。

现在把名片改成交互式的：

```python
print("=== 名片生成器 ===")

name = input("请输入姓名：")
job = input("请输入职业：")
email = input("请输入邮箱：")

print("\n========== 个人名片 ==========")
print("姓名：", name)
print("职业：", job)
print("邮箱：", email)
print("==============================")
```

那个 `\n` 是换行符，让输出空一行，视觉上更清爽。

阿码在旁边看了一眼，突然举手："如果用户输入的是数字呢？`input()` 返回的到底是什么类型？"

好问题。我们来做个实验：

```python
age = input("请输入你的年龄：")
print("你输入的是：", age)
print("它的类型是：", type(age))
```

不管用户输入的是 25 还是"二十五"，`input()` 拿到的永远是**字符串**（string）——也就是一段文字。即使输入的是数字 25，Python 也会把它当成文字 "25" 来处理。

这一点非常重要，后面做计算的时候我们会再遇到。现在先记住：**`input()` 返回的永远是字符串**。

现在你的名片生成器已经可以交互了。但输出还是有点简陋——姓名、职业、邮箱只是简单地罗列。能不能做得好看一点？

## 4. 让输出更优雅

字符串在 Python 里是一种特殊的数据，你可以对它做各种"操作"。

比如**拼接**（concatenation），用加号把两个字符串连起来：

```python
greeting = "你好，" + "世界"
print(greeting)   # 输出：你好，世界
```

但拼接有个麻烦：如果中间要插变量，代码会变得很啰嗦：

```python
name = "李小明"
job = "Python 学习者"

print("姓名：" + name + "，职业：" + job)
```

这还能看。但如果信息多了，加号、引号、变量混在一起，很容易出错。

Python 3.6 之后引入了一种更优雅的写法，叫 **f-string**（formatted string literal，格式化字符串字面量）。只要在字符串前面加个 `f`，就可以在引号里直接用大括号插入变量：

```python
name = "李小明"
job = "Python 学习者"
email = "xiaoming@example.com"

print(f"========== 个人名片 ==========")
print(f"姓名：{name}")
print(f"职业：{job}")
print(f"邮箱：{email}")
print(f"==============================")
```

大括号里的内容会被替换成变量的值。代码清晰多了，而且引号里写什么，输出就长什么样，很直观。

f-string 还支持**格式说明符**。比如控制小数位数：

```python
pi = 3.1415926
print(f"圆周率约等于 {pi:.2f}")   # 输出：圆周率约等于 3.14
```

`:.2f` 表示"保留两位小数的浮点数"。以后处理数据的时候这会很有用。

现在用 f-string 美化一下我们的名片：

```python
print("=== 名片生成器 ===\n")

name = input("请输入姓名：")
job = input("请输入职业：")
email = input("请输入邮箱：")

print(f"\n╔{'═' * 30}╗")
print(f"║{'个人名片':^30}║")
print(f"╠{'═' * 30}╣")
print(f"║  姓名：{name:<20}║")
print(f"║  职业：{job:<20}║")
print(f"║  邮箱：{email:<20}║")
print(f"╚{'═' * 30}╝")
```

这里用到了几个小技巧：
- `^30` 表示居中对齐，占 30 个字符宽度
- `<20` 表示左对齐，占 20 个字符宽度
- `{'═' * 30}` 把字符重复 30 次，画出边框

运行一下，是不是有点专业名片的感觉了？

到目前为止，你的程序运行得都很顺利。但真正的编程——哪怕是写这种小程序——大部分时间其实都在和错误打交道。

## 5. 程序报错了——别慌

小北正在完善他的名片生成器，突然屏幕上跳出一堆红色的字。他心头一紧："完了，是不是我把电脑搞坏了？"

别慌。报错是 Python 在跟你说话，不是惩罚你。它是在说："你刚才的指令我没听懂，请检查一下。"

让我们故意制造几个错误，看看 Python 会说什么。

**错误一：SyntaxError（语法错误）**

```python
print("Hello World!
```

漏了右边的引号。Python 会报：

```
SyntaxError: unterminated string literal (detected at line 1)
```

"unterminated string literal" 意思是"没结束的字符串"。Python 读到行尾还没找到配对的引号，就懵了。

**错误二：NameError（名字错误）**

```python
print(message)
message = "Hello"
```

先使用变量，后定义变量。Python 会报：

```
NameError: name 'message' is not defined
```

Python 从上往下读代码，读到 `print(message)` 的时候，它还没见过 `message` 这个名字。

**错误三：TypeError（类型错误）**

```python
age = "25"
print(age + 5)
```

Python 会报：

```
TypeError: can only concatenate str (not "int") to str
```

你想把字符串 "25" 和数字 5 相加，Python 不知道你想做算术（25+5=30）还是做拼接（"25"+"5"="255"），干脆拒绝执行。

看到报错信息，第一件事是**别慌**。第二件事是**从上到下读**：

1. **错误类型**：SyntaxError、NameError、TypeError……告诉你大概是什么类别的问题
2. **错误描述**：用英文解释具体发生了什么
3. **出错位置**：`line X` 告诉你第几行出的问题

大部分时候，修复错误就是这三步：定位行号、理解错误类型、修改代码。刚开始可能会觉得英文报错很难懂，但常见的就那几种，见多了就习惯了。

老潘路过看了一眼小北的屏幕："报错是 Python 在帮你。如果它默默吞掉错误，给你个奇怪的结果，那才难 debug。"

确实。报错虽然看起来吓人，但它指出了问题所在。学会和报错和平相处，是成为程序员的必修课。

## Git 本周要点

本周必会命令：
- `git init` —— 初始化仓库
- `git status` —— 查看当前状态
- `git add <file>` —— 添加文件到暂存区
- `git commit -m "message"` —— 提交更改
- `git log --oneline` —— 查看提交历史

常见坑：
- 只改文件不提交：下周找不到"当时怎么改的"。至少做 draft + verify 两次提交。
- 提交信息太随意：用动词开头，说明做了什么（如 "add: 名片生成器基础版本"）

## 本周小结（供下周参考）

这一周，你写下了第一行 Python 代码，学会了让程序输出、记忆、与用户交互。你完成了一个可以交互运行的个人名片生成器——从硬编码到变量化，再到交互式输入和格式化输出。最重要的是，你学会了面对报错不要慌，因为调试本身就是编程的一部分。

下周，我们将让程序学会"做选择"——用 `if` 语句让程序变得更智能，比如根据用户的输入给出不同的回应。

## Definition of Done（学生自测清单）

完成本周学习后，请确认你能：

- [ ] 运行 Python 程序，看到预期的输出
- [ ] 使用变量存储至少 3 种不同类型的信息（姓名、年龄、职业等）
- [ ] 编写一个使用 `input()` 获取用户输入的程序
- [ ] 使用 f-string 格式化输出，让内容美观易读
- [ ] 故意制造一个错误，读懂报错信息并修复它
- [ ] 使用 Git 提交至少 2 次（draft + verify）
- [ ] 运行 `python3 -m pytest chapters/week_01/tests -q` 并通过所有测试
